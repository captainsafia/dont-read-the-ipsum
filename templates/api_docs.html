API Documentation

Query from Twitter

The /api/comment/ipsum/twitter endpoint allows you to fetch a random ipsum text generator from an account's tweets or replies to their tweets. The API takes in a value count which represents the number of count_type that you would like returned. Note that count_type can only be char or word, in another value is provided the API will automatically default to word as the count_type. The endpoint also takes a value source which is the account's username without the "@" symbol in the front. The final parameter is tweets_or_replies, which can contain either the values "tweets" or "replies". If it is set to "tweets" the API will generate ipsum from the account's own tweets and if it is set to "replies" the API will generate ipsum from the replies to that account's tweets.

Request

            
                POST /api/comment/ipsum/twitter
                {
                    "count": 280,
                    "count_type": "char",
                    "source": "captainsafia",
                    "tweets_or_replies": "tweets"
                }

Response

                }

Query from Reddit Thread

The /api/comment/ipsum/reddit endpoint allows you to fetch a random ipsum generated from the comments on a Reddit submission. The API takes in a value count which represents the number of count_type that you would like returned. Note that count_type can only be char or word, if another value is provided the API will automatically default to word as the value for count_type. The endpoint also takes a value source which is the short URL for the reddit submission that you wish to generate ipsum from. The short URL will typically look like "https://redd.it/XXXXX" where the XXXXX is the ID of that particular Reddit thread.

Request

            
                POST /api/comment/ipsum/reddit
                {
                    "count": 500,
                    "count_type": "words",
                    "source": "https://redd.it/mateq"
                }

Response

            
                {
                    "text": "on voted You've was guy in will breakthrough for more been the paradox you funny very ever as still into called Will the to you"
                }

Query from New York Times

The /api/comment/ipsum/nyt endpoint allows you to fetch a random ipsum text generator for a New York Times article search query. The API takes in a value count which represents the number of words that you would like returned. The endpoint also takes a value source which is the single word search string. Note that it does not accept multiple words. The API will take the search word and return a jumble of random words from the first paragraphs of articles that match the search word.

                POST /api/comment/ipsum/nyt
                {
                    "count": 280,
                    "source": "obamacare",
                }
